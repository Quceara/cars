# ENCAR Car Landing

Веб-приложение на **FastAPI**: собирает объявления легковых автомобилей с корейского маркетплейса [ENCAR](https://www.encar.com) через официальный JSON API, сохраняет нормализованные данные в JSON и показывает каталог на главной странице с постраничной подгрузкой карточек.

## Возможности

- Запрос к API `https://api.encar.com/search/car/list/mobile` с фильтром по легковым авто (`CarType.A`).
- Сохранение в `data/encar_cars.json` только полей: марка, модель, год, пробег, цена, фото (атомарная перезапись файла).
- Метаданные выгрузки в `data/encar_meta.json` (в т.ч. `total_on_site`, `collected_in_file`, время обновления).
- Полный каталог: сегментированный сбор по фасетам из `iNav` с рекурсивным дроблением запросов и дедупликацией по `Id` (обход лимита «один запрос — не весь каталог»).
- Ежедневное обновление через **APScheduler** (по умолчанию **03:00** локального времени сервера).
- При старте приложения выполняется одно обновление данных, затем запускается планировщик.
- Главная страница: счётчики «всего на сайте» / «всего в файле», карточки подгружаются порциями через `/api/cars`.

## Стек

- Python 3.11+ (рекомендуется; проверяйте совместимость с вашей версией)
- FastAPI, Uvicorn, Jinja2
- APScheduler
- Стандартная библиотека для HTTP (`urllib`)

## Установка и запуск

Рабочая директория — **корень репозитория** (`car-landing`), не папка `app`.

### Windows (PowerShell)

```powershell
cd C:\path\to\car-landing
python -m venv venv
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Откройте в браузере: `http://127.0.0.1:8000`

### Linux / macOS

```bash
cd /path/to/car-landing
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Если `pip` ругается на кодировку `requirements.txt`, пересохраните файл в UTF-8 или установите зависимости вручную по списку пакетов из `requirements.txt`.

## Переменные окружения

| Переменная | Назначение |
|------------|------------|
| `ENCAR_TEST_LIMIT` | Если задано положительное число: парсинг ограничен примерно этим количеством записей (быстрый тест), на сайте отображаются только первые N карточек из файла. Если не задано — полный сегментированный сбор и весь каталог в UI (в пределах файла). |

Пример (PowerShell):

```powershell
$env:ENCAR_TEST_LIMIT=100
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Отключить тестовый режим:

```powershell
Remove-Item Env:ENCAR_TEST_LIMIT
```

## HTTP API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | HTML-лендинг с каталогом |
| GET | `/api/cars?page=1&page_size=24` | JSON: порция карточек, поля `items`, `total_in_file`, `has_more` |
| GET | `/health` | Проверка живости сервиса |
| POST | `/update-now` | Запуск обновления данных вручную (долгая операция при полном каталоге) |

Параметры пагинации: `page` ≥ 1, `page_size` от 1 до 100 (см. константы в `app/main.py`).

## Расписание обновлений

В `app/scheduler.py` задано ежедневное выполнение в **03:00** по локальному времени машины, где крутится процесс. Чтобы сменить время, измените `DAILY_HOUR` и `DAILY_MINUTE`.

## CLI-парсер (без веб-сервера)

Диагностика и выгрузка в файл из корня проекта:

```bash
python app/parser.py --help
python app/parser.py --output data/encar_cars.json
```

По умолчанию без `--max-batches` собирается максимум доступный объём для базового запроса; для ограничения теста: `--max-batches 1`.

## Деплой на сервере

1. Склонируйте/скопируйте проект на сервер.
2. Создайте venv, установите зависимости, запустите из **корня** `car-landing`:

   ```bash
   python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```

3. Откройте порт в firewall и при необходимости настройте reverse proxy (nginx) и HTTPS.
4. Для фонового режима используйте **systemd**, **supervisor** или контейнер; учтите, что первый запуск и `POST /update-now` при полном каталоге могут занимать много времени и трафика.

## Важные замечания

- Данные берутся из публичного API ENCAR; перед коммерческим использованием проверьте **условия использования** сайта и API.
- Объём каталога большой: полная выгрузка создаёт нагрузку на сеть и диск; для демо удобен `ENCAR_TEST_LIMIT`.
- Файлы `data/encar_cars.json` и `data/encar_meta.json` в репозитории по умолчанию игнорируются (см. `.gitignore`).
